# App Review Insights Agent 需求文档

## 1. 文档信息

- 项目名称：App Review Insights Agent
- 文档版本：V1.0
- 编写日期：2026-07-19
- 项目阶段：校招 Homework 本地 Demo 提交版
- 适用范围：App Store 评论分析、JSON/CSV 评论导入、AI 主题发现、版本规划、PRD、测试用例、证据追溯和报告导出

## 2. 项目背景

产品经理人工阅读大量 App Store 评论时存在明显成本：

- 评论数量大，人工筛选耗时。
- 英文评论和中英文混合评论阅读成本高。
- 用户反馈分散，同类问题表达不一致。
- 需求判断经常缺少可追溯的原始证据。
- PRD 和测试用例编写需要反复查阅用户问题。
- 直接使用 AI 可能产生幻觉，生成没有评论证据支撑的主题、需求或测试用例。

本项目希望把评论分析流程做成一个本地可运行工具，让用户能够从真实评论或导入数据出发，生成结构化且可追溯的产品交付物。

## 3. 项目目标

- 快速聚合评论中的主要用户问题。
- 支持用户自定义分析目标，例如订阅付费、稳定性、功能需求等。
- 生成可继续编辑的版本规划、PRD 和测试用例草案。
- 保留从评论到测试用例的完整证据链。
- 降低模型幻觉风险，过滤无效 ID 和无证据结论。
- 支持陌生 App、陌生 JSON/CSV 数据输入。
- 在 API Key 缺失、模型失败或网络异常时保持页面可用，并提供示例缓存结果。

## 4. 非项目目标

当前版本不做：

- 自动修改 App 或业务代码。
- 自动发布版本。
- 模型训练或微调。
- 用户账号系统。
- 云端部署。
- 实时监控全部 App Store 评论。
- 替代人工最终产品决策。
- 多 Agent 编排。
- LangChain 工作流。
- 数据库存储和历史记录。

## 5. 用户角色

### 产品经理

需求：快速判断低评分评论中的关键问题，并形成版本规划和 PRD。

场景：面向某个 App 或导入评论数据，输入“重点分析订阅和付费问题”，查看问题主题、需求和验收标准。

### 用户研究人员

需求：聚合同类反馈，查看原始评论证据，识别冲突反馈。

场景：上传调研整理出的 JSON/CSV 评论，检查主题是否有足够评论支持。

### 测试人员

需求：从 PRD 获得可执行测试用例，并追溯到用户问题。

场景：查看每条测试用例关联的需求、问题和评论原文。

### 面试评审人员

需求：验证项目是否可运行，AI 是否真实参与，结果是否有证据校验。

场景：克隆仓库，运行 Demo，使用示例数据或自己的评论文件进行验收。

## 6. 核心使用场景

- App Store 链接分析：输入 App Store 链接，系统采集美国区评论并分析。
- JSON/CSV 分析：上传本地评论文件，系统复用同一套清洗和 AI 链路。
- 聚焦某个分析目标：用户输入目标，模型 Prompt 明确围绕目标归纳。
- 查看问题证据：每个主题展示支持评论和冲突评论原文。
- 生成版本规划：基于有效问题主题生成版本目标和优先级。
- 生成 PRD：输出需求背景、用户问题、目标、方案、范围、验收标准和风险。
- 生成测试用例：从需求生成具体步骤和预期结果。
- 导出结果：导出 JSON 或 Markdown 报告。
- API 失败时使用示例结果：加载缓存报告并明确标注不是实时模型输出。

## 7. 功能需求

### FR-001 数据来源选择

- 需求描述：用户可以选择 App Store 链接、JSON 文件、CSV 文件或示例数据作为输入。
- 输入：App Store URL、JSON 文件、CSV 文件或示例数据按钮。
- 处理逻辑：App Store 链接和文件上传二选一；示例数据按本地 JSON 处理。
- 输出：`scope.dataSource` 标记当前来源。
- 异常情况：同时提供链接和文件时返回错误；两者都为空时返回错误。
- 验收标准：页面显示当前数据来源，错误输入不会白屏。
- 优先级：P0

### FR-002 App Store 链接解析

- 需求描述：从 App Store 链接中解析 App ID。
- 输入：包含 `/id数字` 的 App Store 链接。
- 处理逻辑：使用正则解析 App ID，并通过美国区评论接口采集评论。
- 输出：`collection.appId`、评论列表和采集页状态。
- 异常情况：普通网页链接或错误 App Store 链接返回可理解错误。
- 验收标准：合法链接可采集；非法链接给出提示。
- 优先级：P0

### FR-003 JSON 导入

- 需求描述：支持上传 JSON 评论数组。
- 输入：`id`、`rating`、`title`、`content`、`author`、`date`、`version` 字段。
- 处理逻辑：解析 JSON 数组，映射到内部 `RawReview`。
- 输出：可进入清洗流程的评论集合。
- 异常情况：非法 JSON 或非数组结构显示错误。
- 验收标准：`sample_data/reviews.json` 可成功导入并分析。
- 优先级：P0

### FR-004 CSV 导入

- 需求描述：支持上传 CSV 评论文件。
- 输入：表头 `id,rating,title,content,author,date,version`。
- 处理逻辑：使用 Papa Parse 解析，映射到内部 `RawReview`。
- 输出：可进入清洗流程的评论集合。
- 异常情况：CSV 格式错误显示可理解错误。
- 验收标准：`sample_data/reviews.csv` 可成功导入并分析。
- 优先级：P0

### FR-005 数据清洗

- 需求描述：过滤空内容、异常评分和重复评论。
- 输入：原始评论列表。
- 处理逻辑：校验 `rating` 是否为 1-5；合并标题与正文判断空内容；按作者、标题、正文和评分去重。
- 输出：`cleaning` 报告和 `reviews` 清洗后列表。
- 异常情况：全部为空或全部异常时继续返回清洗报告，不白屏。
- 验收标准：重复评论、空评论和非法评分被计数。
- 优先级：P0

### FR-006 基础统计

- 需求描述：展示评论数量、平均评分、评分分布、低评分比例和版本分布。
- 输入：清洗后的评论。
- 处理逻辑：确定性计算统计值。
- 输出：`metrics`。
- 异常情况：无有效评论时显示 0 或空状态。
- 验收标准：页面基础统计区域可读。
- 优先级：P0

### FR-007 分析目标

- 需求描述：用户可以填写分析目标或使用快捷目标。
- 输入：文本框和快捷按钮。
- 处理逻辑：空目标使用默认目标；目标传入 `/api/analyze` 并进入模型 Prompt。
- 输出：`scope.goal`。
- 异常情况：空字符串不报错，使用默认目标。
- 验收标准：结果区显示“本次分析目标”。
- 优先级：P0

### FR-008 AI 主题发现

- 需求描述：从低评分评论中动态发现用户问题主题。
- 输入：清洗后 1-2 星评论，以及最多 20 条 4-5 星冲突候选评论。
- 处理逻辑：服务端调用 OpenAI-compatible API，要求返回严格 JSON。
- 输出：`issueDiscovery.themes`。
- 异常情况：API Key 缺失、模型失败、非法 JSON 均显示错误。
- 验收标准：主题标题由模型动态生成，不能依赖固定关键词分类。
- 优先级：P0

### FR-009 证据校验

- 需求描述：校验模型返回的评论 ID、问题 ID 和证据关系。
- 输入：模型输出和当前有效评论/主题/需求。
- 处理逻辑：删除虚构 ID；过滤无有效证据的结果；由代码计算数量。
- 输出：已校验结果和 warnings。
- 异常情况：模型虚构 ID 时不白屏，显示 warning。
- 验收标准：无效 ID 不进入最终展示。
- 优先级：P0

### FR-010 版本规划

- 需求描述：基于有效主题生成版本计划。
- 输入：`issueDiscovery.themes`。
- 处理逻辑：模型生成候选内容，代码生成 `VP-xxx` 并校验 `includedIssueIds`。
- 输出：`productPlanning.versionPlans`。
- 异常情况：无有效主题时跳过。
- 验收标准：版本规划展示版本名称、目标、优先级、问题和理由。
- 优先级：P0

### FR-011 PRD 生成

- 需求描述：基于有效主题生成结构化 PRD。
- 输入：已校验主题和分析目标。
- 处理逻辑：模型生成候选需求，代码生成 `REQ-xxx`，过滤无证据或无验收标准需求。
- 输出：`productPlanning.requirements`。
- 异常情况：证据不足时显示“证据不足，暂不生成需求”。
- 验收标准：每条需求包含来源问题和来源评论。
- 优先级：P0

### FR-012 测试用例生成

- 需求描述：基于有效 PRD 生成结构化测试用例。
- 输入：`productPlanning.requirements`。
- 处理逻辑：模型生成候选测试用例，代码生成 `TC-xxx`，校验需求、问题、评论关联。
- 输出：`testGeneration.testCases`。
- 异常情况：空需求、空步骤、空预期结果、虚构 ID 均过滤。
- 验收标准：高优先级需求至少具备正常流程和异常/边界用例。
- 优先级：P0

### FR-013 追溯关系

- 需求描述：展示评论到测试用例的完整链路。
- 输入：主题、需求和测试用例。
- 处理逻辑：按需求计算 `评论 R-xxx → 问题 F-xxx → 需求 REQ-xxx → 测试用例 TC-xxx`。
- 输出：`traceability`。
- 异常情况：链路缺失时显示 warning。
- 验收标准：展示评论原文和追溯完整率。
- 优先级：P0

### FR-014 报告导出

- 需求描述：导出完整 JSON 和 Markdown 报告。
- 输入：当前页面分析结果。
- 处理逻辑：前端根据结果对象生成 Blob 下载。
- 输出：`.json` 或 `.md` 文件。
- 异常情况：未分析时不展示导出按钮。
- 验收标准：导出文件包含主题、PRD、测试用例和追溯链，不包含 API Key。
- 优先级：P1

### FR-015 示例缓存

- 需求描述：提供演示兜底结果。
- 输入：点击 `查看示例分析结果`。
- 处理逻辑：服务端读取 `sample_outputs/example-analysis.json`。
- 输出：标记为缓存的完整分析结果。
- 异常情况：文件缺失时显示错误。
- 验收标准：页面标注“示例缓存结果，不是本次实时模型输出”。
- 优先级：P1

### FR-016 错误提示

- 需求描述：所有关键失败都必须可理解。
- 输入：错误链接、错误文件、API Key 缺失、模型失败、非法 JSON 等。
- 处理逻辑：API 返回错误信息，前端展示错误框或阶段状态。
- 输出：可读错误提示和兜底入口。
- 异常情况：页面不得白屏。
- 验收标准：用户能知道失败原因和下一步操作。
- 优先级：P0

## 8. 非功能需求

- 可用性：单页工作台完成输入、分析、查看和导出。
- 可维护性：AI 主题、PRD、测试、导入和追溯分别位于独立模块。
- 安全性：API Key 只在服务端读取，不进入前端和导出文件。
- 可解释性：每个主要结论都保留来源 ID 和原文。
- 可测试性：核心校验逻辑有 Vitest 自动化测试。
- 兼容性：支持 Windows、macOS、Linux 常规 Node.js 环境。
- 性能要求：当前 Demo 以中小规模评论集为目标，App Store 采集页数有限制。
- 错误恢复：模型或网络失败时提供明确错误和示例缓存入口。
- 隐私和密钥保护：不提交 `.env`、`.env.local`，导出报告不包含环境变量。

## 9. 数据需求

评论数据字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 否 | 缺失时生成导入来源 ID |
| rating | number | 是 | 必须为 1-5 |
| title | string | 否 | 评论标题 |
| content | string | 是 | 评论正文，空内容会删除 |
| author | string | 否 | 缺失时使用 anonymous |
| date | string | 否 | 可被 `Date` 解析，缺失时使用稳定兜底时间 |
| version | string | 否 | 缺失时使用 unknown |

处理规则：

- 支持中英文混合数据。
- `rating` 为 0、6 或无法转成数字时会在清洗阶段过滤。
- 空评论内容会在清洗阶段过滤。
- 重复评论按作者、标题、正文和评分去重。
- App Store 数据如果缺少版本号，版本统计显示为 `unknown`。

## 10. AI 输出要求

所有 AI 阶段必须返回结构化 JSON。

主题字段：

- `issueId`
- `title`
- `summary`
- `severity`
- `confidence`
- `supportingReviewIds`
- `conflictingReviewIds`

版本规划字段：

- `versionName`
- `objective`
- `priority`
- `includedIssueIds`
- `rationale`

PRD 字段：

- `title`
- `background`
- `userProblem`
- `productGoal`
- `proposedSolution`
- `inScope`
- `outOfScope`
- `acceptanceCriteria`
- `priority`
- `risks`
- `sourceIssueIds`
- `sourceReviewIds`

测试用例字段：

- `title`
- `requirementId`
- `sourceIssueIds`
- `sourceReviewIds`
- `priority`
- `preconditions`
- `steps`
- `expectedResult`
- `testType`

禁止虚构证据。任何模型返回的 ID 都必须能在当前数据结构中找到，否则删除或过滤。非法 JSON 会使对应阶段进入 error。

## 11. 状态与异常流程

- 正常完成：所有阶段依次完成，展示结果和导出按钮。
- API Key 未配置：基础统计可运行，AI 阶段显示明确错误，提供示例缓存入口。
- 模型请求失败：对应 AI 阶段显示错误，页面不白屏。
- 模型返回非法 JSON：显示非法 JSON 错误，不展示伪造结果。
- App Store 获取失败：API 返回采集错误，页面显示错误信息。
- 上传文件格式错误：JSON/CSV 解析失败时显示文件格式错误。
- 没有低评分评论：AI 主题发现跳过，后续 PRD 和测试也跳过。
- 有问题但证据不足：过滤无证据结果，并显示“证据不足”。
- 缓存结果展示：页面明确标记缓存结果不是实时模型输出。

## 12. 验收标准

- 数据来源选择：App Store、JSON、CSV 和示例数据均可选择。
- 清洗统计：非法评分、空内容、重复评论能被识别。
- AI 主题：模型输出必须经过 JSON 解析和 ID 校验。
- 版本规划：只引用有效 `issueId`。
- PRD：只引用有效问题和支持评论，验收标准不能为空。
- 测试用例：只引用有效需求、问题和评论，步骤和预期结果不能为空。
- 追溯关系：可按需求查看评论原文和完整链路。
- 导出报告：JSON/Markdown 文件可下载，且不含密钥。
- 异常兜底：无 API Key 或模型失败时页面不白屏。

## 13. 优先级划分

### P0：必须完成

- 数据来源选择
- App Store / JSON / CSV 输入
- 评论清洗和基础统计
- 分析目标
- AI 主题发现
- 证据校验
- PRD 和测试用例生成
- 追溯完整率
- 错误提示

### P1：重要

- 版本规划
- 报告导出
- 示例缓存结果
- README 和中文文档
- 面试验收报告

### P2：后续优化

- 人工编辑确认
- 历史分析记录
- 多 App 对比
- 更细粒度的 token 管理
- 在线部署和权限系统

## 14. 风险与应对

- 外部接口不稳定：支持 JSON/CSV 导入和示例缓存兜底。
- 模型幻觉：所有 ID 和证据关系由代码校验。
- Token 消耗：限制采集页数和冲突候选评论数量。
- 长评论上下文限制：当前先截断部分输入，后续可做分批聚合。
- 模型输出格式不稳定：要求 JSON，并用 Zod 校验。
- 数据不足：允许跳过主题、PRD 或测试生成。
- 评论冲突：使用高评分评论作为冲突候选，不把冲突评论当需求证据。
- API 服务不可用：页面展示错误，并提供缓存示例结果。
